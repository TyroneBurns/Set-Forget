How the HMM strategy works

This app uses a Hidden Markov Model style regime engine to estimate whether the market is in a bull, bear, or chop state. It reads recent closed candles, measures momentum and ATR-based volatility, then updates probabilities as new candles arrive.

The engine combines two ideas:

  • Emissions: how much the latest candle behaviour looks like a bullish, bearish, or choppy market
  • Transitions: how likely the market is to stay in the same regime or switch into another one

The result is three probabilities. The strongest one becomes the current regime. A signal only becomes tradable when confidence and spread are strong enough.

What the new quality score does

v3 adds a simple ML-style ranking layer on top of the HMM. It does not retrain a model yet, but it scores each signal using:

  • current HMM confidence
  • bull vs bear spread
  • recent paper-trade performance
  • chop penalty

This creates a signal quality score out of 100. Better signals rise to the top of the watchlist.

How adaptive thresholds work

The app also adjusts the confidence threshold slightly based on recent paper-trade returns. If recent trading has gone well, the threshold loosens a bit. If recent trading has gone badly, the threshold tightens so the app becomes more selective.

How machine learning could improve this later

A fuller machine learning layer could learn which combinations of pair, timeframe, volatility, and time of day tend to work best. That could improve pair ranking, signal acceptance, and filtering of false breakouts.

This build gives you the groundwork and explanation in-app, while staying honest about what is currently live.

How performance is shown

The overview shows realised paper-trade returns over the last 1 day and 7 days, so you can quickly judge whether the app has been effective recently.
